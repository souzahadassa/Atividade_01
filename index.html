GUIA COMPLETO: Git e GitHub

O que é o Git

O Git é um sistema de controle de versão distribuído, criado por Linus Torvalds, que permite que desenvolvedores acompanhem as alterações no código e colaborem de forma eficiente em projetos de software.

O que é o GitHub

O GitHub é uma plataforma de hospedagem de código-fonte com controle de versão usando Git. Ele facilita o trabalho em equipe, revisão de código, documentação e colaboração aberta ou privada.

Diferenças entre Git e GitHub

  • Git é a ferramenta de controle de versões.
  • GitHub é uma plataforma baseada na nuvem que usa o Git.
  • Com o Git você versiona localmente, com o GitHub você compartilha e colabora online.

Principais comandos Linux para usar no Git Bash

  • ls - Lista arquivos e diretórios.
  • cd - Navega entre diretórios.
  • mkdir - Cria uma nova pasta.
  • rm - Remove arquivos.
  • clear - Limpa o terminal.

Comandos do readme.md no repositório aula_git

  • git init - Inicia um repositório Git.
  • git add . - Adiciona todos os arquivos.
  • git commit -m "mensagem" - Salva uma versão com uma mensagem.
  • git push - Envia as mudanças para o GitHub.

Como criar um repositório no GitHub

  1. Entre em github.com.
  2. Clique em New ou "+" > New repository.
  3. Dê um nome, escolha visibilidade, e clique em Create repository.

Como ligar um projeto local a um repositório GitHub

git init
git remote add origin https://github.com/usuario/repositorio.git
git add .
git commit -m "Primeiro commit"
git push -u origin master

Como gerar uma versão com Git

Cada versão é um commit. Para gerar uma:

git add .
git commit -m "Versão 1.0 - Adiciona funcionalidades iniciais"

Como enviar da máquina para o GitHub

Depois de comitar, envie com:

git push origin master

Se for o primeiro envio, use também:

git rem
